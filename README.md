# java-kanban-practicum — RESTful Kanban Task Manager

Java, REST API, GSON, JUnit, HTTP Server, No Frameworks

## О проекте

**Kanban Task Manager** — backend-приложение для управления задачами в формате Kanban-доски.  
Проект реализован на **чистой Java без использования фреймворков** и предназначен для изучения принципов построения REST API, работы с HTTP, сериализации данных и тестирования.

Приложение позволяет создавать задачи, объединять их в эпики, управлять подзадачами, отслеживать историю просмотров и контролировать пересечение задач по времени.

## Основные сущности

В системе реализованы три типа задач:

- **Task** — обычная задача
- **Epic** — крупная задача, объединяющая подзадачи
- **Subtask** — подзадача, входящая в состав эпика

Каждая задача имеет:
- идентификатор;
- название и описание;
- статус выполнения;
- время начала и продолжительность (опционально).

## Реализованная функциональность

### Управление задачами
- создание задач, эпиков и подзадач;
- обновление задач;
- удаление задач по идентификатору;
- получение списка всех задач.

### Работа с эпиками
- автоматический расчёт статуса эпика на основе подзадач;
- привязка подзадач к эпику;
- удаление эпика вместе со всеми подзадачами.

### История просмотров
- хранение истории просмотра задач;
- история не содержит дубликатов;
- при удалении задачи автоматически удаляется из истории.

### Приоритизация задач
- хранение задач в отсортированном виде по времени начала;
- проверка пересечения задач по времени;
- предотвращение логических конфликтов в расписании.

## HTTP API

Приложение предоставляет REST API для управления задачами.

### Tasks
- `GET /tasks/task` — получение списка задач
- `POST /tasks/task` — создание или обновление задачи
- `GET /tasks/task/{id}` — получение задачи по id
- `DELETE /tasks/task/{id}` — удаление задачи

### Epics
- `GET /tasks/epic` — получение списка эпиков
- `POST /tasks/epic` — создание или обновление эпика
- `GET /tasks/epic/{id}` — получение эпика по id
- `DELETE /tasks/epic/{id}` — удаление эпика

### Subtasks
- `GET /tasks/subtask` — получение списка подзадач
- `POST /tasks/subtask` — создание или обновление подзадачи
- `GET /tasks/subtask/{id}` — получение подзадачи по id
- `DELETE /tasks/subtask/{id}` — удаление подзадачи
- `GET /tasks/subtask/epic/{epicId}` — получение подзадач эпика

### Дополнительные эндпоинты
- `GET /tasks/history` — история просмотра задач
- `GET /tasks/prioritized` — список задач, отсортированных по приоритету

## Хранение данных

Реализовано несколько вариантов хранения:

- **InMemoryTaskManager** — хранение в памяти
- **FileBackedTaskManager** — сохранение задач в файл
- **HTTP Task Manager** — удалённое хранение через KVServer

### KVServer
- хранит данные в формате key-value;
- доступ осуществляется по API-токену;
- используется для сохранения состояния Task Manager.

## Архитектура

Проект построен с чётким разделением ответственности:

- модели задач (Task, Epic, Subtask);
- менеджеры задач;
- менеджер истории;
- HTTP-сервер;
- сериализация/десериализация через GSON.

Приложение не использует Spring или другие фреймворки, вся логика реализована вручную.

- тестирование сохранения и восстановления состояния.
